# ✅ Repository Optimization Checklist

## For Every Repository

### Identity & Discovery
- [ ] Repository name: lowercase, hyphenated, descriptive
- [ ] Description: 1 sentence, no trailing period, under 100 characters
- [ ] Topics: 6–10 relevant tags (language + framework + domain)
- [ ] Website URL: link to live demo or portfolio
- [ ] License: MIT (unless reason for otherwise)

### Documentation
- [ ] README.md: complete (see template in BRANDING_GUIDE.md)
- [ ] README has badges (build status, license, tech stack)
- [ ] README has screenshots or demo GIF
- [ ] README has "Getting Started" section with setup commands
- [ ] README has "Architecture" or "How it works" section
- [ ] CONTRIBUTING.md: for any public project you want PRs on
- [ ] CHANGELOG.md: for published packages or tools

### Code Quality Signals
- [ ] .gitignore: complete and project-appropriate
- [ ] No credentials committed (check with git-secrets or similar)
- [ ] Folder structure is logical and documented
- [ ] Consistent code style (ESLint/Prettier configured)
- [ ] At least some test coverage

### GitHub Features
- [ ] Releases: tag milestones as v1.0.0, v1.1.0, etc.
- [ ] Issues: open a few planned features as issues
- [ ] Projects: use GitHub Projects for active repos

## GitHub SEO Quick Wins

1. **Keywords in bio:** "Full Stack Developer | AI Engineer | React | Node.js | LLMs"
2. **Keywords in README:** Technologies named explicitly (not just as icons)
3. **Active commits:** At least 4–5 commits per week minimum
4. **Stars:** Star repos you actively learn from (signals your interests)
5. **Following:** Follow engineers you respect in the AI/FullStack space

---

## Accessibility Improvements

- [ ] All images in README have `alt` text
- [ ] Badge labels are descriptive (not just colored dots)
- [ ] Color is not the only way to convey information
- [ ] Links have descriptive text (not "click here")
- [ ] Tables have header rows

---

## Performance Recommendations

- [ ] Large images compressed before adding to repo
- [ ] SVG assets used instead of PNGs where possible
- [ ] No binary files in the main branch history
- [ ] Use CDNs for external resources (shields.io, github-readme-stats)

---

## Final Polish Checklist

Before considering your profile "launch-ready":

- [ ] Read the README as a recruiter would — does it answer the 7 key questions?
- [ ] Mobile-test all linked live demos
- [ ] All badge URLs return 200 (not broken links)
- [ ] Typing animation reads smoothly (test at readme-typing-svg.demolab.com)
- [ ] Snake animation workflow runs successfully
- [ ] GitHub stats cards render (sometimes needs a public email)
- [ ] Profile photo is high quality
- [ ] No spelling errors anywhere
- [ ] LinkedIn and GitHub bios are consistent in story
