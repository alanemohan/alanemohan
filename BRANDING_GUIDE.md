## Commit Message Convention (Conventional Commits)

```
feat:     New feature
fix:      Bug fix
docs:     Documentation only
style:    Formatting, no logic change
refactor: Code restructure, no feature/fix
perf:     Performance improvement
test:     Tests
chore:    Build, tooling, dependencies

Examples:
  feat: add streaming response to AI assistant
  fix: resolve context overflow in chat history
  docs: update README with architecture diagram
  refactor: extract LLM client into service layer
```

---

## Branch Naming Convention

```
feature/feature-name        → New features
fix/bug-description         → Bug fixes
docs/section-name           → Documentation
refactor/component-name     → Code refactoring
chore/tooling-task          → Tooling, dependencies

Examples:
  feature/mcp-integration
  fix/api-key-validation
  docs/readme-overhaul
  refactor/ai-client-service
```

## Open Source Strategy

### Phase 1 — Getting Started
1. Find repos you use daily (e.g., shadcn/ui, vercel/ai, langchain)
2. Start with **documentation fixes** — low risk, high visibility
3. Move to **good first issues** — labeled in most repos
4. Submit 1 quality PR per month minimum

### Phase 2 — Building Reputation
1. Fix bugs in repos related to your AI/Full Stack stack
2. Write and submit feature proposals with implementation plans
3. Review other contributors' PRs (shows leadership)
4. Blog about your contributions on GitHub Discussions or Dev.to
